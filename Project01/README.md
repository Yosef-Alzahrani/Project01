## Getting Started

1. Run `npm install`
2. Run `npm run dev`

## Backend API (SQLite + Login/Register)

- Run API only: `npm run dev:api` (defaults to http://localhost:5000)
- Run web + API together: `npm run dev:all`
- SQLite DB file: `server/data/app.sqlite` (auto-created)

### API Endpoints

- `POST /api/auth/register` { email, password, name? }
- `POST /api/auth/login` { email, password }
- `GET /api/auth/me` (Authorization: Bearer TOKEN)
- `POST /api/predict` (multipart/form-data, field name: image)
- `POST /api/predictions` (JSON body; store prediction result without image)
- `GET /api/model/metrics`
- `GET /api/analytics`

## Using a Real AI Model (ONNX)

This project supports running a **real ONNX model locally in the browser** (no image upload).

1. Put your ONNX model file here: `public/models/wound_classifier.onnx`
2. Ensure the ONNX Runtime WASM files exist at: `public/onnxruntime/` (auto-copied on `npm install` via `postinstall`)
3. Enable it:
	- Set `VITE_USE_LOCAL_ONNX=true` in `.env`
	- (Optional) set `VITE_ONNX_MODEL_URL=/models/wound_classifier.onnx`

When enabled:
- The wound image is processed **locally** in the browser.
- Only the prediction result is sent to `/api/predictions` so the dashboard analytics can use SQLite.
